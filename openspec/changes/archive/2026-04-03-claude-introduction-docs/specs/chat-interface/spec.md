## ADDED Requirements

### Requirement: Document Artifacts feature

CHAT.md SHALL explain the Artifacts feature, including what types of content can be generated as artifacts (code, SVG, HTML, documents) and how to interact with them.

#### Scenario: Engineer uses Artifacts for code generation

- **WHEN** an engineer reads the Artifacts section
- **THEN** they SHALL understand how to generate, preview, and copy artifact content from Claude.ai

### Requirement: Document Projects and Memory

CHAT.md SHALL explain Claude Projects, including how to create projects, add files, and how project memory persists across conversations.

#### Scenario: Engineer sets up a project

- **WHEN** an engineer reads the Projects section
- **THEN** they SHALL be able to create a project and understand how context is maintained across sessions

### Requirement: Document file upload capabilities

CHAT.md SHALL describe supported file types for upload (images, PDFs, code files, documents) and their use cases.

#### Scenario: Engineer uploads a document for analysis

- **WHEN** an engineer reads about file uploads
- **THEN** they SHALL know which file types are supported and what Claude can do with them

### Requirement: Document conversation management

CHAT.md SHALL cover conversation history, branching, and sharing capabilities in Claude.ai.

#### Scenario: Engineer shares a conversation

- **WHEN** an engineer wants to share a Claude conversation with a colleague
- **THEN** CHAT.md SHALL explain how to share and the privacy implications
