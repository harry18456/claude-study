## ADDED Requirements

### Requirement: Document Claude model families

GENERAL.md SHALL describe the three Claude model families (Haiku, Sonnet, Opus) with their intended use cases and capability trade-offs.

#### Scenario: Engineer reads about model selection

- **WHEN** an engineer reads GENERAL.md
- **THEN** they SHALL be able to identify which model suits their use case based on speed, cost, and capability requirements

### Requirement: Document access methods

GENERAL.md SHALL list all ways to access Claude: Claude.ai web interface, Claude Code CLI, and Claude API.

#### Scenario: Engineer identifies entry point

- **WHEN** an engineer wants to start using Claude
- **THEN** GENERAL.md SHALL provide a clear map of available access methods and their primary use cases

### Requirement: Document pricing tiers

GENERAL.md SHALL describe the available plans (Free, Pro, Team, Enterprise) and their key differences relevant to engineering teams.

#### Scenario: Engineer evaluates plan for team

- **WHEN** an engineer needs to recommend a plan for team adoption
- **THEN** GENERAL.md SHALL provide sufficient information to make a recommendation
